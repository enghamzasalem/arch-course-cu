# Container Descriptions – LLM API Wrapper System

## 1. Proxy Gateway
**Technology:** Python FastAPI  
**Responsibility:**  
Acts as the entry point for all client requests.  
It receives prompts from applications and forwards them to external LLM APIs such as OpenAI or Gemini.  
It also collects metadata about each request and sends it to the tracking service.

---

## 2. Tracking Service
**Technology:** Python worker service  
**Responsibility:**  
Records metadata for every LLM request, including prompt size, tokens used, latency, cost, and response status.  
The data is stored in the Logs Database for further analysis.

---

## 3. Monitoring Service
**Technology:** Prometheus / Monitoring API  
**Responsibility:**  
Aggregates system metrics such as request rate, error rate, latency, and token usage.  
Provides dashboards or APIs to monitor the health and performance of the system.

---

## 4. Anomaly Detection Service
**Technology:** Python ML service  
**Responsibility:**  
Analyzes usage patterns and monitoring metrics to detect unusual behaviors such as spikes in requests, abnormal latency, or unexpected cost increases.

---

## 5. Logs Database
**Technology:** PostgreSQL or MongoDB  
**Responsibility:**  
Stores detailed request logs and usage metadata generated by the tracking service.

---

## 6. Metrics Database
**Technology:** Time-series database (Prometheus / InfluxDB)  
**Responsibility:**  
Stores aggregated monitoring metrics used for dashboards and anomaly detection.
